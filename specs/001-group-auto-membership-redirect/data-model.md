# Data Model: Group Auto-Membership and Redirect on Creation

**Feature**: 001-group-auto-membership-redirect  
**Date**: 2026-04-23

---

## No Schema Changes Required

This feature requires **no database schema changes**. All necessary tables and
relationships already exist. This document records the relevant existing entities
for implementation reference.

---

## Entities Involved

### Group

Existing table: `groups`

| Field | Type | Notes |
|-------|------|-------|
| `id` | `uuid` | PK, auto-generated |
| `name` | `text` | Provided by user on creation |
| `code` | `text` | Unique 8-char invite code, auto-generated by DB trigger |
| `created_at` | `timestamptz` | Auto-set |

**Behaviour on creation**: INSERT into `groups` with `name`. The `code` field is
populated automatically.

### Membership

Existing table: `memberships`

| Field | Type | Notes |
|-------|------|-------|
| `id` | `uuid` | PK |
| `group_id` | `uuid` | FK ‚Üí `groups.id` |
| `user_id` | `uuid` | FK ‚Üí `profiles.id` (= `auth.users.id`) |
| `created_at` | `timestamptz` | Auto-set |

**Behaviour on group creation**: INSERT one row with `group_id = new group's id` and
`user_id = authenticated user's id`. This must succeed before the redirect fires.

### Failure Modes

| Failure point | State after | Recovery |
|---------------|-------------|----------|
| Group insert fails | Nothing created | User sees error, retries |
| Membership insert fails | Group exists without creator as member | User sees error; group is orphaned (acceptable for v1 per research.md) |
| `setCurrentGroup` fails | Group + membership exist, cookie not set | Redirect still fires but page returns 404 (same bug as current) ‚Äî this call should be treated as critical |

## `currentGroup` Cookie

Not a database entity but part of the data flow.

| Key | Type | Set by | Read by |
|-----|------|--------|---------|
| `currentGroup` | `string` (UUID) | `setCurrentGroup(groupId)` | `getCurrentGroup()` |

**Required invariant**: The `currentGroup` cookie MUST be set to the new group's ID
before `redirect()` is called; otherwise the group page returns 404.
